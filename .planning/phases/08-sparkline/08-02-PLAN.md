    **Step 1 — 개발 서버 확인 및 실행:**

    `mcp__playwright__browser_navigate`로 `http://localhost:3000/dashboard` 접속.
    (로그인이 필요하면 Credentials Provider로 이메일만 입력하여 로그인)

    **Step 3 — Daily 탭 스파크라인 확인:**
    `mcp__playwright__browser_snapshot`으로 페이지 구조 캡처.
    KPI 카드 영역에 SVG 요소(AreaChart)가 5개 렌더링되는지 확인한다.
    각 카드에 스파크라인 선이 표시되고 기존 달성률/델타 요소가 정상인지 확인한다.

    **Step 4 — Weekly 탭 전환 후 확인:**
    Weekly 탭 버튼 클릭 → 스냅샷으로 weekly 데이터 기반 스파크라인 5개 확인.
    매출 카드의 달성률 + 프로그레스 바 + 스파크라인이 함께 정상 표시되는지 확인한다.

    **Step 5 — 다크 모드 전환 확인 (가능한 경우):**
    헤더의 테마 토글 버튼 클릭 → 다크 모드 전환 후 스파크라인 색상이 변경되는지 스냅샷 확인.
    CSS 변수 `var(--chart-1)` 방식이므로 테마 전환 시 색상이 자동으로 변경되어야 한다.

    **실패 시 대응:**
    - 스파크라인이 보이지 않으면: `kpi-card.tsx`의 조건부 렌더링 조건(`sparklineData.length >= 2`) 및 데이터 전달 여부 재확인
    - 빌드 에러가 있으면: 08-01 플랜으로 돌아가 오류 수정
    - 레이아웃이 깨지면: `ResponsiveContainer` height 및 `minWidth={0}` 설정 확인
