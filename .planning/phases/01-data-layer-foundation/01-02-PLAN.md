    ```typescript
    const DAILY_HEADERS = {
      date: "일자",
      revenue: "매출",
      profit: "손익",
      usageHours: "이용시간",
      usageCount: "이용건수",
      utilizationRate: "가동률",
    } as const;

    const WEEKLY_HEADERS = {
      week: "주차",
      revenue: "매출",
      profit: "손익",
      usageHours: "이용시간",
      usageCount: "이용건수",
      utilizationRate: "가동률",
      weeklyTarget: "목표",
    } as const;
    ```

    **buildColumnIndex 함수:**
    헤더 배열을 받아 헤더명 → 인덱스 Map을 반환한다.
    빈 헤더 셀은 Map에 추가하지 않는다.

    **parseKoreanNumber 함수 (DATA-03 핵심):**
    CONTEXT.md 결정을 정확히 구현한다:
    - value가 undefined, null, 빈 문자열 → 0 반환
    - 공백 trim → ₩ 제거 → % 제거 → 콤마(,) 제거 → Number() 변환
    - 변환 결과가 NaN이면 console.warn 후 null 반환
    - 반환 타입: number | null

    **safeNumber 헬퍼:**
    parseKoreanNumber 결과가 null이면 fallback(기본 0)을 반환한다.
    반환 타입: number

    **parseDailySheet 함수 (DATA-01):**
    - 매개변수: rows: string[][]
    - rows가 비어있거나 행이 1개 미만이면 빈 배열 반환
    - rows[0]으로 buildColumnIndex 호출하여 colIndex 생성
    - DAILY_HEADERS의 각 값에 대해 colIndex에 없으면 console.warn 경고
    - rows.slice(1)에서 빈 행(모든 셀이 빈 문자열) 필터링
    - 각 row에서 헤더 이름으로 값을 가져와 DailyRecord 객체 생성
    - date는 string 타입 그대로 (.trim() 적용), 나머지 숫자 필드는 safeNumber() 적용
    - open-ended range와 함께 사용하므로 rows 끝의 빈 행 제거 필수

    **parseWeeklySheet 함수 (DATA-02):**
    - parseDailySheet와 동일한 패턴, WEEKLY_HEADERS 사용
    - weeklyTarget 필드를 포함하여 WeeklyRecord 객체 생성

    **getTeamDashboardData 함수 (DATA-04, UX-02):**
    - 반환 타입: Promise<TeamDashboardData>
    - isGoogleSheetsConfigured() === false → mockTeamDashboardData 즉시 반환
    - isGoogleSheetsConfigured() === true → try/catch 블록:
      - Promise.all로 두 시트를 병렬 fetch:
        fetchSheetData(`${DAILY_SHEET}!A:F`)
        fetchSheetData(`${WEEKLY_SHEET}!A:G`)
      - 각 fetch 결과가 null이면 해당 mock 배열로 대체 (per-sheet fallback은 허용)
      - return { daily, weekly, fetchedAt: new Date().toISOString() }
      - catch(error): console.error 후 { ...mockTeamDashboardData, fetchedAt: new Date().toISOString() } 반환
      (CONTEXT.md 결정: API 실패 시 전체 mock 폴백)

    googleapis SDK는 Next.js fetch 캐싱을 사용하지 않으므로 기본적으로 매 요청마다 새로 실행된다.
